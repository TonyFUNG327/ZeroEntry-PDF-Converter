# GUI Upload Entry Design

Status: deferred. Current V.2 direction follows the ZeroEntry_PDF converter V.14 folder-based flow, where users place source files into folders such as `BB`. This note is retained only as a future GUI reference and should not drive the current implementation.

## Goal

Provide a simple local upload entry for the Accounting AI Engine workflow.

The GUI should guide users through three stages:

1. Opening balance import
2. Bank/Cash/Manual Journal source import
3. Generate GL, TB, BS, and IS

## Required Upload Slots

### Stage 1: Opening Balance

Required:

- Prior-year Trial Balance workbook
- Closing date
- Opening date

Output:

- Opening balance review workbook
- Opening balance JSON payload

### Stage 2: Current Period Records

Required:

- Bank record workbook

Optional:

- Cash record workbook
- Completed Manual Journals workbook

Generated by system:

- Manual Journals input template
- COA reconciliation workbook / report

Output:

- Bank journal lines
- Cash journal lines
- Manual journal lines, if supplied
- COA mapping warnings

### Stage 3: Accounting Reports

Required:

- Confirmed opening balance payload
- Bank journal payload

Optional:

- Cash journal payload
- Manual journal payload
- Sample report workbook for comparison

Output:

- GL
- TB
- BS
- IS
- Comparison report, if sample report supplied

## UI Flow

Recommended local GUI pages:

1. `Project Setup`
   - Company name
   - Period start / end
   - Year end date
   - Output folder

2. `Upload Files`
   - Opening TB
   - Bank record
   - Cash record
   - Manual Journals
   - Sample final report

3. `Validate`
   - File detection status
   - Visible sheets detected
   - Hidden sheets skipped
   - COA mismatch warnings
   - Debit/Credit balance checks

4. `Generate`
   - Run Stage 1
   - Run Stage 2
   - Run Stage 3
   - Open output folder

## Recommended Implementation Path

For V.2, use a simple local web app:

- Backend: Python FastAPI
- Frontend: single HTML page with file inputs
- Processing: call existing parser/build scripts
- Storage: per-project folders under `projects/<project_name>/`

Initial routes:

- `GET /`
- `POST /api/project`
- `POST /api/upload/opening-tb`
- `POST /api/upload/bank-record`
- `POST /api/upload/cash-record`
- `POST /api/generate/manual-journals-template`
- `POST /api/run/opening`
- `POST /api/run/stage2`
- `POST /api/run/full-period`

## Safety Rules

- Never overwrite uploaded source files.
- Store generated files under a project-specific output folder.
- Show warnings before allowing final report generation.
- Hidden sheets should be skipped by default.
- Account mapping conflicts should block Stage 3 unless user confirms.
