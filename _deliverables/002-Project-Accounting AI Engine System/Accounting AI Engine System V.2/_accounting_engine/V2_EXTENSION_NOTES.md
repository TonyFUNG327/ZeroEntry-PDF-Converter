## COA Reconciliation

Added prototype:

- `coa_reconciliation.py`

Purpose:

- Compare `Chart of Accounts` across input files.
- Detect:
  - same account description with different IDs
  - same account ID with different descriptions

Test result:

- Compared I026 Bank record COA against Cash record template COA.
- Found 13 mapping issues.

This confirms that Stage 1/2/3 should not rely blindly on Account ID alone.

Recommended Stage 2 behavior:

1. Read COA from all uploaded/generated workbooks.
2. Generate reconciliation report.
3. Block Stage 3 if mapping conflicts are unresolved.
4. Allow user-confirmed mappings to override source differences.

## Folder-Based Operation Flow

V.2 should follow the same operating style as ZeroEntry_PDF converter V.14:

- User places source workbooks into the designated input folder, such as `BB`.
- Program reads files from the folder and writes intermediate JSON payloads plus review Excel files to the output/staging folders.
- No GUI upload entry is required at this stage.

Recommended input classes:

- Opening TB from prior year closing values
- Bank record workbook
- Cash record workbook
- Optional Manual Journals workbook generated by the system for user entry
- Optional sample TB/GL/BS/IS workbook for conversion comparison
